## 0. 四量模型(实证自原版 infantry.txt)

一个营/师有**四个独立量**, 我之前混淆了:

| 量 | 原版字段 | 性质 | 当前实现 | 真实机制 |
|---|---|---|---|---|
| 组织度 Org | `max_organisation` | 当前战斗状态 | ✅ org | 归零→撤退(不消失) |
| HP | `max_strength` | 固定血量 | ✅ strength(但理解错) | 影响属性衰减、经验损失; **不是装备充足度** |
| 人力 Manpower | `manpower` | 库存资源(兵员) | ❌ 缺失 | 国家征召池, 独立增援 |
| 装备 Equipment | `need` | 库存资源(武器) | ✅ equipment_held | 工厂生产, 独立增援 |

### 关键修正(我之前的错误)
- ❌ 把 `strength`(HP) 当成"装备充足度"
- ❌ 认为只有装备影响有效属性
- ✅ 真实: 装备充足度 和 人力充足度 是两个独立比例, 共同影响有效属性
- ✅ 真实: 战斗 HP 损失 → 同时消耗装备和人力

### 有效属性公式(修正)
```
装备比 = equipment_held / equipment_need
人力比 = manpower_held / manpower_need
综合充足度 = min(装备比, 人力比)   # 木桶效应, 短板决定
有效属性 = 面板属性 × 综合充足度
```

### 战斗退出的两种方式(用户澄清 — 关键!)

| 退出类型 | 触发条件 | 后果 |
|---|---|---|
| **撤退 Retreat** | 组织度归零, **HP 未归零** | 师撤到邻近己方省份, **存活**, 脱离战斗恢复 org 后可再战 |
| **歼灭 Annihilation** | **HP 归零** | 师从世界**彻底消失**, 撤销番号, 装备/人力全损失 |
| 装备条 | 不直接决定退出 | 只影响有效属性(缺装备→攻防下降), 归零师仍留战场挨打 |

两种结局的本质区别:
- 撤退 = 可恢复的失败(师还在, 能回来)
- 歼灭 = 永久损失(番号没了, 对应原版 shatter, 给敌方 35 XP)

**当前实现的错误**: 我把"组织度归零"当唯一退出条件, 且都按"移出战斗"处理。
**修正方向**:
- org 归零 + HP>0 → 撤退(移出战斗, 师保留, 待行军机制后撤邻省)
- HP 归零 → 歼灭(从 world.divisions 删除, 给敌方加 XP)

## 1. 完整陆战循环(我之前只做了中间切片)

```
[行军] 师从A省移动到B省(速度/地形/被拦截)
   ↓
[进入战斗] 省份宽度有限, 超出的师进增援队列
   ↓
[互殴] 伤害结算(✅已做) — 多方向/宽度/堆叠
   ↓ 一方组织度耗尽
[撤退] 破阵师向后省份移动(受追击伤害, 不是消失)
   ↓
[增援补位] 队列中的师按 REINFORCE_CHANCE 概率进场
   ↓
[脱离战斗] 组织度按恢复速度回升(士气恢复, 受地形/补给影响)
   ↓
[损耗补充] 装备+人力从仓库补充(✅装备已做, 人力缺失)
   ↓ 循环
```

## 2. 缺失环节清单(按依赖排序)

| 优先级 | 环节 | 依赖 | 说明 |
|---|---|---|---|
| **P0** | 人力模型 | 无(地基) | 加 manpower_need/held, 综合充足度=min(装备,人力) |
| **P1** | 组织度恢复 | 无 | 非战斗师每小时回升 org(基础恢复率) |
| **P2** | 增援队列 | 省份宽度模型 | 战斗宽度限制, 排队进场, REINFORCE_CHANCE |
| **P2** | 行军 | 省份邻接 | 师在省间移动, 速度/到达判定 |
| **P3** | 撤退 | 行军 | 破阵师后撤而非消失, 受追击 |
| **P3** | 省份宽度/多方向 | 地形 | 战斗宽度限制多师进场 |

## 4. 当前实现的修正点(M4b' 要改)

1. Division 加 `manpower_need` / `manpower_held`
2. Country 加 `manpower_pool`(征召池) + 每周增长
3. `equipment_ratio` 改为 `supply_ratio = min(装备比, 人力比)`
4. 战斗 HP 损失 → 同时消耗装备和人力
5. 增援机制同时补装备和人力
6. 组织度恢复: 非战斗师每小时 `org += recovery_rate`(如 max_org × 5%/天)
7. create_division 支持 manpower 参数
